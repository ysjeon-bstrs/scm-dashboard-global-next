# Contributing

Thanks for contributing to SCM Dashboard Global Next.

## Good First Areas

- Typed data models for inventory, logistics movements, and Excel imports
- Tests for `src/lib/scm-dashboard` transforms and allocation logic
- Source adapter examples that use mock or sample data
- Accessibility and responsive improvements for dashboard components
- Documentation for running, deploying, and adapting the dashboard

## Pull Request Checklist

Before opening a pull request:

1. Run `npm run lint`
2. Do not commit real credentials, customer data, business data, or private database details
3. Include a short verification note in the PR description
4. Keep changes scoped to one behavior or documentation improvement

## Security

Never include OAuth client secrets, service-role keys, database passwords, API keys, real exports, or internal infrastructure details in issues or pull requests.
